## 概览

饿了么的商品定制分为三层：

| 层级 | 英文 | 示例 | 用途 |
|------|------|------|------|
| 规格 (Specs) | specs | 大杯 / 中杯 / 小杯 | 影响价格的基础选项 |
| 属性 (Attrs) | attrs | 冰 / 热 / 常温 | 不影响价格的偏好选项 |
| 加料 (Ingredients) | ingredients | 加一份浓缩 / 换燕麦奶 | 可能影响价格，有互斥规则 |

## Specs — 规格

每个商品可能有一个或多个规格维度：

```json
{
  "specs": [
    {
      "name": "规格",
      "options": ["大杯", "中杯", "小杯"]
    }
  ]
}
```

选择不同规格会改变 `item_id` 和 `sku_id`。下单时传入对应规格的 ID 即可。

## Attrs — 属性

不影响价格的定制选项：

```json
{
  "attrs": [
    {
      "name": "温度",
      "options": ["冰", "热"]
    },
    {
      "name": "甜度",
      "options": ["正常糖", "半糖", "少糖", "无糖"]
    }
  ]
}
```

## Ingredients — 加料

最复杂的定制层，包含**互斥规则**：

```json
{
  "ingredients": [
    {
      "group_name": "浓缩",
      "required": true,
      "options": [
        {
          "id": "ig_0_0",
          "name": "标准浓缩",
          "price": 0.0,
          "item_id": "11111",
          "sku_id": "22222",
          "package_id": "33333"
        },
        {
          "id": "ig_0_1",
          "name": "双倍浓缩",
          "price": 2.0,
          "item_id": "11112",
          "sku_id": "22223",
          "package_id": "33334",
          "excludes": ["ig_1_0", "ig_1_1"]
        }
      ],
      "default_id": "ig_0_0"
    },
    {
      "group_name": "萃取方式",
      "required": false,
      "options": [
        {
          "id": "ig_1_0",
          "name": "热萃",
          "price": 0.0,
          "item_id": "21111",
          "sku_id": "32222",
          "package_id": "43333"
        },
        {
          "id": "ig_1_1",
          "name": "冷萃",
          "price": 3.0,
          "item_id": "21112",
          "sku_id": "32223",
          "package_id": "43334"
        }
      ],
      "default_id": "ig_1_0"
    }
  ]
}
```

### 互斥规则

每个选项的 `excludes` 数组列出了与它**不兼容**的选项 ID：

```
选择「双倍浓缩」(ig_0_1) → excludes: ["ig_1_0", "ig_1_1"]
→ 「萃取方式」组的所有选项被排除
→ 该组不可选
```

ID 格式为 `ig_{组索引}_{选项索引}`，稳定且可预测。

<Warning>
  互斥规则是**单向**的：A 排除 B 不意味着 B 排除 A。需要逐一检查每个选中选项的排除列表。
</Warning>

### 典型场景

**星巴克链式排除：**
```
浓缩 → 萃取方式 → 份数
选择「双倍浓缩」排除整个「萃取方式」组
「萃取方式」的选项又可能排除「份数」组的某些选项
```

**互斥选择：**
```
可换糖 ↔ 甜度
选择「换糖」排除所有甜度选项
选择任意甜度排除「换糖」选项
```

## Default Ingredients — 默认选择

为方便 AI Agent 快速下单，菜单返回了预计算的 `default_ingredients`：

```json
{
  "default_ingredients": [
    {
      "item_id": "11111",
      "sku_id": "22222",
      "package_id": "33333",
      "buy_amount": 1
    }
  ]
}
```

这是系统自动选择每个必选组的第一个非互斥选项后的结果。Agent 可以直接将此字段传入下单接口，无需理解互斥逻辑。

<Tip>
  对于大多数下单场景，直接使用 `default_ingredients` 是最简单的方式。只有用户明确要求定制时才需要手动选择。
</Tip>
