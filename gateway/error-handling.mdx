## 统一错误格式

所有错误响应遵循统一格式：

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "人类可读的错误描述"
  }
}
```

## 错误码参考

### 认证错误 (401)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `AUTH_REQUIRED` | 缺少 `Authorization` 请求头 | 添加 `Bearer clw_...` 头 |
| `AUTH_INVALID` | API Key 无效或已被禁用 | 检查 Key 是否正确，联系管理员确认状态 |
| `USER_TOKEN_REQUIRED` | 缺少 `X-User-Token` 请求头 | 通过 SMS 绑定流程获取 Token |
| `USER_TOKEN_INVALID` | User Token 无效或已过期 | 重新绑定用户（Token 默认 90 天有效）|

### 客户端错误 (400)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `SMS_CODE_INVALID` | 验证码错误或已过期 | 验证码 10 分钟有效，重新发送 |

### 限流错误 (429)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `SMS_COOLDOWN` | 60 秒内重复发送验证码 | 等待冷却时间结束 |
| `RATE_LIMITED` | 超过请求频率限制 | 降低请求频率（下单 10次/分钟，其他 60次/分钟）|

### 资源错误 (404)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `SHOP_NOT_FOUND` | 商家 ID 不存在 | 检查 shop_id 格式（应为 E 前缀）|

### 服务端错误 (500)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `ORDER_FAILED` | 订单创建或预览失败 | session_id 可能已过期（10 分钟），重新预览 |

### 上游错误 (502)

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| `UPSTREAM_ERROR` | 外卖平台 API 返回错误 | 稍后重试，检查参数是否正确 |

## HTTP 状态码总结

| 状态码 | 含义 |
|--------|------|
| 200 | 请求成功 |
| 400 | 客户端参数错误 |
| 401 | 认证失败 |
| 404 | 资源不存在 |
| 429 | 请求频率超限 |
| 500 | 服务端内部错误 |
| 502 | 上游 API 不可用 |
